MovieTracker

A Java(FX) Desktop Application to track and manage watched movies

Goals of Project

  • Practice learned content from Lessons
  • Understand and Review Program from the 5 main Perspectives
  • Practice Git Workflow
  • Create Program I would actually use to make my life easier

Overview of Application

  • Data Mgmt App / CRUD Program
  • When watching huge amount of Movies its hard to keep track and keep the memories fresh
  • This Program tries to adress this problem by creating a simple way to store watched movies in a CSV file

Architecture and Design

  • Follows MVC Structure
  • Model: Representing Movie Data(main.java and movie.java)
  • View: Creates UI and handles User Interaction(all fxml files)
  • Controller: Handles user input(View) and updates Model accordingly(all controller classes)

Complexity

Interactions

  • Dependencies exist between classes for handling user input, displaying data in the UI, and managing movie data
  • User interactions, such as clicking buttons to create or edit movies
  • The Goal is to drive the flow of control within the application
